# TypeScript পরিচিতি এবং বেসিক টাইপস (Introduction to TypeScript)

এই ফাইলটিতে TypeScript-এর বেসিক ধারণা, JavaScript-এর সাথে পার্থক্য, বেসিক ডাটা টাইপস এবং Type Annotation বনাম Type Inference সম্পর্কে বিস্তারিত আলোচনা করা হয়েছে।

---

## ১. TypeScript আর JavaScript-এর পার্থক্য এবং কেন TypeScript ব্যবহার করব?

**JavaScript (JS)** হলো একটি **Dynamically Typed** স্ক্রিপ্টিং ল্যাঙ্গুয়েজ। এর মানে হলো, কোড রান করার আগে ভ্যারিয়েবলের টাইপ চেক করা হয় না। কোড রান করার সময় যেকোনো ভুল টাইপ থাকলে এরর দেয় বা অপ্রত্যাশিত আচরণ করে।

**TypeScript (TS)** হলো JavaScript-এর একটি **Superset** (যা মূলত JS-এর ওপর অতিরিক্ত সুবিধা যোগ করে)। এটি একটি **Statically Typed** ল্যাঙ্গুয়েজ। এর মানে হলো কোড রান করার আগেই (Compilation-এর সময়) টাইপ সংক্রান্ত ভুলগুলো ধরা পড়ে।

### মূল পার্থক্যসমূহ:

| বৈশিষ্ট্য | JavaScript (JS) | TypeScript (TS) |
| :--- | :--- | :--- |
| **টাইপিং** | Dynamically Typed (রানটাইমে টাইপ নির্ধারিত হয়) | Statically Typed (কম্পাইল টাইমে টাইপ চেক হয়) |
| **ত্রুটি সনাক্তকরণ** | রানটাইমে ভুল ধরা পড়ে, যা প্রোডাকশনে বাগ তৈরি করতে পারে। | কোড লেখার সময় বা কম্পাইল করার সময়ই ভুল ধরা পড়ে। |
| **ডেভেলপার এক্সপেরিয়েন্স** | অটো-কমপ্লিশন (IntelliSense) বা সাজেশন তুলনামূলক কম কার্যকর। | অসাধারণ অটো-কমপ্লিশন, রিফ্যাক্টরিং এবং সাজেশন পাওয়া যায়। |
| **ব্রাউজার সাপোর্ট** | ব্রাউজার সরাসরি রান করতে পারে। | ব্রাউজার সরাসরি রান করতে পারে না; কম্পাইল করে JS-এ রূপান্তর করতে হয়। |

### কেন আমরা TypeScript ব্যবহার করব?
1. **আগেভাগেই বাগ ধরা (Early Bug Detection):** কোড রান করার আগেই টাইপের অমিল বা বানান ভুল (Typo) সনাক্ত করা যায়।
2. **অসাধারণ ইন্টেলিসেন্স (Rich IDE Support):** আপনি যখন কোনো অবজেক্ট বা ফাংশন ব্যবহার করবেন, তখন কোড এডিটর নিজেই আপনাকে বলে দেবে তার ভেতর কী কী প্রোপার্টি বা প্যারামিটার আছে।
3. **সহজ রিফ্যাক্টরিং (Easy Refactoring):** বড় কোডবেসে কোনো ফাংশন বা প্রোপার্টির নাম পরিবর্তন করা অত্যন্ত সহজ ও নিরাপদ হয়ে যায়।
4. **সহজ পাঠযোগ্যতা (Self-Documenting Code):** কোড দেখেই বোঝা যায় কোন ফাংশন কী ধরনের ডেটা ইনপুট নেয় এবং কী রিটার্ন করে।

## ২. TypeScript-এর Basic Types (উদাহরণসহ)

TypeScript-এ বিভিন্ন ধরনের প্রিমিটিভ এবং নন-প্রিমিটিভ টাইপ রয়েছে। নিচে প্রধান ৪টি টাইপ আলোচনা করা হলো:

### ক. `string`
যেকোনো টেক্সট বা লেখার জন্য `string` টাইপ ব্যবহার করা হয়। এটি সিঙ্গেল কোট (`'`), ডাবল কোট (`"`) অথবা ব্যাকটিক (`` ` ``) দিয়ে লেখা যায়।
```typescript
let username: string = "Sahadat Hossen";
let greeting: string = `Hello, ${username}!`;
```

### খ. `number`
সব ধরণের সংখ্যার জন্য (পূর্ণসংখ্যা বা দশমিক) `number` টাইপ ব্যবহার করা হয়।
```typescript
let age: number = 25;
let price: number = 99.99;
let hex: number = 0xf00d; // Hexadecimal
```

### গ. `boolean`
শুধুমাত্র `true` অথবা `false` মান রাখার জন্য `boolean` টাইপ ব্যবহার করা হয়।
```typescript
let isLoggedIn: boolean = true;
let hasPermission: boolean = false;
```

### ঘ. `array`
একই টাইপের একাধিক ডেটার লিস্ট রাখার জন্য `array` ব্যবহার করা হয়। এটি দুইভাবে ডিফাইন করা যায়:
* `type[]` সিনট্যাক্স ব্যবহার করে (সবচেয়ে বেশি ব্যবহৃত)।
* `Array<type>` (Generics) সিনট্যাক্স ব্যবহার করে।

```typescript
// পদ্ধতি ১: string-এর অ্যারে
let programmingLanguages: string[] = ["JavaScript", "TypeScript", "Python"];

// পদ্ধতি ২: number-এর অ্যারে (Generics ব্যবহার করে)
let scores: Array<number> = [95, 88, 100];
```

## ৩. Type Annotation বনাম Type Inference

### Type Annotation (টাইপ অ্যানোটেশন):
ডেভেলপার যখন নিজে নিজে ম্যানুয়ালি কোনো ভ্যারিয়েবল, ফাংশন প্যারামিটার বা রিটার্ন ভ্যালুর টাইপ নির্দিষ্ট করে দেয়, তাকে **Type Annotation** বলে। এর জন্য কোলন (`:`) ব্যবহার করা হয়।

* **কখন ব্যবহার করবেন?** যখন আপনি আগে থেকে মান অ্যাসাইন করছেন না, অথবা কোডকে একদম স্পষ্ট ও ডকুমেন্ট উপযোগী করতে চান।

```typescript
// ম্যানুয়ালি টাইপ বলে দেওয়া হচ্ছে
let country: string; 
country = "Bangladesh";

function addNumbers(a: number, b: number): number {
    return a + b;
}
```

### Type Inference (টাইপ ইনফ্যারেন্স):
যখন আমরা কোনো ভ্যারিয়েবলে সরাসরি মান (value) অ্যাসাইন করি, তখন TypeScript বুদ্ধিমানভাবে নিজে থেকেই বুঝে নেয় ভ্যারিয়েবলটির টাইপ কী হবে। একেই বলে **Type Inference**। আপনাকে কষ্ট করে টাইপ লিখতে হয় না।

* **কখন ব্যবহার করবেন?** সাধারণ ভ্যারিয়েবল ডিক্লেয়ারেশনের সময় যখন সরাসরি মান দিয়ে দেওয়া হয়, তখন টাইপ না লেখাই উত্তম এবং এটি কোডকে পরিষ্কার রাখে।

```typescript
// TypeScript নিজে থেকেই বুঝে নিয়েছে যে 'message' এর টাইপ string
let message = "Welcome to TypeScript"; 

// যদি আমরা পরে message-এ সংখ্যা অ্যাসাইন করতে চাই, TS এরর দেবে:
// message = 100; // Error: Type 'number' is not assignable to type 'string'.
```

### সহজ পার্থক্য:
* **Annotation:** "আমি নিজে বলে দেব এটি কী টাইপ।" (`let count: number = 10;`)
* **Inference:** "TypeScript নিজে বুঝে নিক এটি কী টাইপ।" (`let count = 10;` -> TS বুঝে নিয়েছে এটি `number`)
